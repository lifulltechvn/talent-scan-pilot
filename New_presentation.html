🧠 AI Parse — Claude Haiku + tool_use

Structured Output: CV → JSON có schema cố định

Cách hoạt động

  1. Text đã masked PII gửi cho Claude Haiku
  2. Dùng tool_use để ép output theo schema
  3. Claude trả về JSON: name, skills, experience, education, insight...
  4. Server validate + normalize (skill aliases, confidence score)
  5. Inject PII thật trở lại sau khi AI trả kết quả

🎛️ Prompt Tuning — Chiến lược chi tiết

1. tool_use + tool_choice forced

Ép Claude điền đúng form, không trả text tự do

tool_choice: {type: "tool", name: "save_cv_data"} // → Claude PHẢI gọi function này // → Output luôn đúng schema // → Không hallucinate thêm field lạ

2. XML delimiters chống injection

CV có thể chứa "Ignore instructions..." → bọc trong tag

<CV_CONTENT> {nội dung CV đã mask PII} </CV_CONTENT> // Claude hiểu: trong tag = DATA, không phải lệnh

3. temperature=0

Cùng CV gọi 2 lần → cùng kết quả. HR tin tưởng scoring nhất quán.

4. System prompt ngắn (tiết kiệm token)

system: "Parse the CV into structured data. Be concise in insight fields (1-2 sentences)." // Chỉ 20 tokens. Schema đã define trong tool_use. // Prompt dài 500 tokens × 1000 CV = $0.5 lãng phí

5. Post-validation (cv_validator)

AI trả "five" thay vì 5? skills là string? → Normalize hết

// "five" → 5, "nodejs" → "Node.js" // skills string → split thành array // Tính confidence: 0.0 → 1.0 // HR biết data đáng tin cỡ nào

6. Two-tier model (tiết kiệm 40%)

Task cần vision → Sonnet ($3). Task text đơn giản → Haiku ($1)

OCR ảnh scan → BEDROCK_MODEL_SONNET (cần vision) Parse CV text → BEDROCK_MODEL_HAIKU (text thôi) Scoring → BEDROCK_MODEL_HAIKU (có rubric rõ) // 60% calls dùng Haiku → tiết kiệm 40% chi phí

🔒 Security — Bảo mật PII

AI không bao giờ thấy thông tin cá nhân thật

PII Protection Flow

  1. Regex extract — tìm email, phone, CCCD, URL, address
  2. Mask text — thay bằng [EMAIL], [PHONE], [CCCD]
  3. Gửi masked text cho AI — AI chỉ thấy placeholder
  4. AI trả structured data
  5. Inject PII thật trở lại — sau khi AI xong

Prompt Injection Guard

Detect patterns: "ignore previous instructions", "you are now...", XML tags injection → sanitize + log warning

⚡ Quá trình khó khăn & Cách giải quyết

Thách thức gặp phải

1. OCR tiếng Việt

CV scan chất lượng kém, pytesseract sai nhiều → Chuyển sang Claude Sonnet Vision — chính xác hơn hẳn

2. LLM output không consistent

JSON thiếu field, format lỗi → Dùng tool_use để force schema + post-validation

3. Scoring bias

LLM đánh giá không nhất quán → Hybrid: Rule 70% (deterministic) + LLM 30% (insight)

 

4. Chi phí AI cao ban đầu

Sonnet cho mọi thứ → tốn → Two-tier: Haiku cho 60% tasks đơn giản, giảm chi phí ~3x

5. Bedrock throttling khi batch 100 CVs

Rate limit → Exponential backoff retry + ThreadPoolExecutor (max 10 workers) + Semaphore + WebSocket progress realtime

6. Skill matching thiếu chính xác

"nodejs" ≠ "Node.js" → Skill Normalizer (alias dictionary 50+ entries) + semantic cosine matching (Titan 1024-dim)

✨ Tính năng AI nổi bật

🧩 Smart Pool

Khi tạo JD mới → tự động match top 50 ứng viên cũ bằng pgvector cosine similarity. Talent Pool "sống" — không bỏ sót ai.

🔍 Duplicate Detection

2 lớp: MD5 file hash (exact match → block) + Name ILIKE fuzzy search (warn HR trước khi tạo). API check-duplicate riêng.

📅 Self-Schedule

Gửi link token-based cho ứng viên tự chọn slot phỏng vấn. Block T7/CN + giờ quá khứ. Auto reminder 24h + 30min.

🛡️ Injection Guard

Detect prompt injection trong CV (attacker nhúng "ignore instructions" vào CV). Sanitize bằng XML delimiter + log cảnh báo.

🔗 Matching — Cosine + Keyword

Thuật toán

// Matching formula match_score = 0.6 × cosine(CV_vector, JD_vector) + 0.4 × keyword_overlap // Scoring formula FINAL = 0.7 × Rule_Score + 0.3 × LLM_Score // Rule Score breakdown Rule = Skill×30% + Cosine×25% + Exp×20% + Edu×15% + Lang×10% // Classification ≥80 → 🥇 Gold (auto-prioritize) 50–79 → 🥈 Silver (HR review) <50 → 🥉 Bronze (Talent Pool, auto re-match)

Embedding Strategy

  • Amazon Titan V2 — 1024-dim, multilingual
  • CV embedding: skills + experience text (first 500 chars)
  • JD embedding: title + description + required_skills
  • pgvector: cosine distance operator <=>
  • Smart Pool: khi tạo JD → query top 50 candidates gần nhất
